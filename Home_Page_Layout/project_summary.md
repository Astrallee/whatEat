# 「今天吃什么」产品设计原型文档

## 一、产品概述

### 1.1 产品定位

「今天吃什么」是一款针对选择困难症用户的日常饮食决策工具，通过随机抽取和智能推荐，帮助用户快速决定每日菜单。

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| 首页桌板 | 今日菜品展示与管理 |
| 一键摇签 | 随机抽取菜品 |
| 生成菜单 | 确定最终菜单方案 |
| 菜库 | 系统菜谱与私房菜管理 |
| 历史记录 | 过往菜单回顾 |
| 收藏夹 | 喜爱的菜品收藏 |

### 1.3 技术栈

- **框架**: React + TypeScript + Vite
- **路由**: React Router
- **UI组件**: shadcn/ui + Radix UI
- **样式**: Tailwind CSS
- **数据存储**: localStorage
- **布局**: 移动端优先，`max-w-md` 居中

## 四、数据模型

### 4.1 localStorage 数据结构

| Key | 类型 | 描述 |
|-----|------|------|
| `todayBoard` | `Record<string, string[]>` | 今日桌板，按分类存储 |
| `historyBoard` | `HistoryRecord[]` | 历史记录列表 |
| `favorites` | `string[]` | 收藏的菜品名称列表 |
| `myDishes` | `Dish[]` | 用户私房菜列表 |
| `decideCount` | `number` | 已决定吃饭次数 |
| `userAvatar` | `string` | 头像 Base64 数据 |
| `todayMenu` | `string[]` | 当前生成的菜单 |

### 4.2 数据结构定义

```typescript
// 菜品
interface Dish {
  name: string;
  tags: string[];
  category: string;  // 荤菜/素菜/汤/主食/甜品
  cuisine: string;  // 川菜/粤菜/家常菜...
  group?: string;   // 自定义分组(私房菜)
}

// 历史记录
interface HistoryRecord {
  id: string;
  date: string;
  timeType: string;  // 早餐/午餐/晚餐
  dishes: {
    name: string;
    category: string;
    tags: string[];
  }[];
}

// 用户信息
interface UserInfo {
  avatar: string;
  nickname: string;
  signature: string;
  decideCount: number;
}
```

## 六、边界情况处理

### 6.1 数据边界

| 场景 | 处理 |
|-----|------|
| 桌板为空 | 显示「一键摇签」 |
| 桌板已满 | 显示「生成今日菜单」 |
| 历史记录为空 | 显示「暂无历史记录」 |
| 收藏为空 | 显示「暂无收藏」 |
| 私房菜为空 | 显示「还没有添加私房菜」 |

### 6.2 操作边界

| 场景 | 处理 |
|-----|------|
| 菜品已存在于桌板 | 提示「该菜品已在桌板中」 |
| 菜品已收藏 | 心标显示已收藏状态 |
| 删除最后一条历史 | 允许，decideCount 不减少 |
| 无 decideCount 但有历史 | 从历史长度初始化 |

### 6.3 异常处理

| 场景 | 处理 |
|-----|------|
| localStorage 不可用 | 降级为内存状态 |
| 图片裁剪失败 | 提示用户重试 |
| 路由参数无效 | 跳转首页 |

## 七、交互规范

### 7.1 通用规范

- 移动端优先设计，最大宽度 417px 居中
- 所有可点击元素至少有 44px 点击区域
- 弹窗使用 Dialog 组件，带遮罩层
- 底部操作使用 Sheet 组件
- 图标使用 lucide-react

### 7.2 动画规范

- 页面切换: 无动画 (SPA)
- 弹窗: Radix UI 内置动画
- 摇签动画: CSS + 定时器
- 点击反馈: `active:scale-95` 或背景色变化

### 7.3 反馈规范

- 操作成功: Toast 提示或 Alert
- 操作失败: Alert 提示错误信息
- 删除操作: 先 confirm 确认

## 八、开发注意事项

### 8.1 已修复问题

1. **Home 页面同步**: 添加 visibilitychange 监听，保持与 localStorage 同步
2. **删除持久化**: 删除操作后同步更新 localStorage
3. **菜品详情**: Library 页面点击菜品名称区域打开详情，+号仅添加
4. **决定计数**: decideCount 从历史初始化，仅随生成菜单增加
